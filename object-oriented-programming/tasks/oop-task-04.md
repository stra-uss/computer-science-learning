#### Exercício 04

📍​ __Âmbito:__
Atividade Conceitual: No desenvolvimento de sistemas de software, compreender os pilares da Programação Orientada a Objetos (POO).

🎯 __Objetivo:__
O objetivo desta atividade, baseando-se nas atividades práticas anteriores, é conceituar herança, polimorfismo e a realação entre os objetos, a fim de aplicá-los em outros contextos de programação de software.

📝 - __Descrição:__

  No desenvolvimento de sistemas de software, os pilares da Programação Orientada a Objetos (POO) são aplicados para garantir modularidade, reaproveitamento de código e facilidade de manutenção.
  
  No código Python da pizzaria, esses conceitos foram utilizados para estruturar as regras de negócio de precificação e o relacionamento entre as entidades (Cliente, Pizza e Pedido).

  Com base no código-fonte das tarefas anteriores, responda aos itens a seguir:

- Enunciado
- A) Herança e Polimorfismo:
  - Explique como a Herança e o Polimorfismo foram aplicados no modelo das pizzas.
  -
  - Em sua resposta, diferencie o comportamento da classe PizzaTradicional e da classe PizzaEspecial em relação à superclasse Pizza, citando os elementos do código que evidenciam esses conceitos.

- B) Relações entre Objetos:
   - Analise a relação existente entre as classes Pedido e Cliente. Trata-se de uma Composição ou de uma Associação/Agregação?
   
   - Justifique sua resposta com base na forma como o objeto Cliente é manipulado no construtor da classe Pedido e no bloco de execução (__main__).
